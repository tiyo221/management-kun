# questions — わからないこと（個人学習バックログ）（モジュール個別仕様）

統合上の位置づけ・共通マスタとの関係・固有データの所在を定める。共通の取り決め（アーキテクチャ・データ規格・名寄せ・デザイン）は [`spec.md`](../../spec.md)（共通仕様）を正とする。

- localStorage: `mk:module:questions:v1`
- ゾーン: **自分**
- 新規モジュール（旧ツールなし。「モジュールを増やして育てる」思想の実践第1弾。Issue #32）

## 役割
自分の「わからないこと」を可視化し、`未解決 → 調査中 → 解決` のループで潰していく個人向けの学習バックログ。skills（ピープル＝他人の能力の可視化）とは役割が異なり、自分ゾーンに置く。

このモジュールは性質の違う2面を持つ（Issue #81）:

- **未解決 / 調査中** ＝ バックログ（片づけるもの）。
- **ナレッジ** ＝ 解決した質問のうち「後で読んでも分かる答え」を残したもの。貯めて何度も引く資産。

解決＝バックログを閉じることは、必ずしもナレッジ化を意味しない。**「どう解決したか」を再利用できる形で残したものだけがナレッジ**になる（＝質問が答えを得てナレッジへ"昇格"する）。

## 共通マスタ関係
People / Project マスタは**使わない**（todo と同じく個人用途）。`tags` は自由文字列で、将来 Tech Radar 等と接続する布石とする（現状は名寄せしない）。

## 固有データ
- `items[]`: `{ id, title, detail, status, tags[], resolvedNote, createdAt, updatedAt, resolvedAt }`。`mk:module:questions:v1`。
  - `status` = `open`（未解決）/ `investigating`（調査中）/ `resolved`（わかった）。
  - `resolvedAt` は `resolved` 化時に設定、`resolved` から戻すと `null`。
  - `resolvedNote`（解決内容／回答）は任意テキスト。未設定（旧データ）でも壊れない（検索・表示とも空扱い。[`spec.md`](../../spec.md) §4.5 後方互換。追加専用フィールドのため version は据え置き）。
- モジュール内部 ID は `q_<epoch>_<rand>`（[`spec.md`](../../spec.md) §4.7）。

## ナレッジの定義
- **ナレッジ ＝ `status === "resolved"` かつ `resolvedNote`（答え）が非空**。判定は `isKnowledge(it)`。
- 答えを残さず閉じただけの resolved（note 空）は**ナレッジではない**（ただの「閉じた」。棚に出さない＝B案・純度優先）。この状態は解決の主導線（答え必須）では生じず、編集モーダルの手動遷移か CSV 取込（`ステータス=わかった` かつ `わかったこと` 空）でのみ生じる。ナレッジ／未解決／調査中のどのタブにも出ず「全て」タブでのみ見えるため、CSV で一括投入すると埋もれやすい点に注意（行に「答えを書く」導線を出して昇格を促す）。
- `counts().knowledge` はナレッジ件数（答えありの resolved）。`knowledge(search)` はナレッジだけをキーワード絞り込みして返す純関数。

## ビュー・導線（ナレッジ再利用）
- **ステータスタブ**は 全て / 未解決 / 調査中 / **ナレッジ**。「ナレッジ」タブは答えありの解決済みのみを **Q→A カード**で表示する（取り消し線は使わない。線は「片づけたタスク」の作法でありナレッジには合わない）。
- **検索対象**はタイトル・詳細・タグに加え `resolvedNote`（答え）も含む。答え本文のキーワードからナレッジを引ける（`filtered()` / ナレッジ専用の `knowledge(search)`）。
- **解決＝ナレッジ化の主導線**: 未解決／調査中の行の「解決」ボタン → 「どう解決した？」モーダルで**答えを必須入力**（空なら閉じられない＝資産化を強制）→ `resolve(id, note)` で `resolved` 化（`resolvedAt` 設定）。保存後は「ナレッジ」タブへ遷移する。
- 答えなしで閉じた resolved は行に「答えを書く」導線を出し、答えを足せばナレッジへ昇格できる。

## CSV
共通契約（[`spec/import-migration.md`](../import-migration.md) §4.6.2）に沿う。列は `タイトル, 詳細, ステータス, タグ, わかったこと`。

- `タイトル` は必須（空の行はスキップ・件数警告）。
- `ステータス` は key（`open` / `investigating` / `resolved`）または日本語ラベル（未解決 / 調査中 / わかった）を寛容解釈する。不明・空は `open`。
- `タグ` は空白またはカンマ区切り（techstack と同形）。
- 取込は全置換。`createdAt` / `updatedAt` は取込時刻で再生成し、`resolvedAt` は `resolved` のとき取込時刻・それ以外は `null`。
- ラウンドトリップ（出力→取込）が成立する（UTF-8 BOM・ステータス key/ラベル両対応・タグ分割）。
